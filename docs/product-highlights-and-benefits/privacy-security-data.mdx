## Saving Code Snippets in the Cloud
The pieces cloud is entirely opt-in:
- Authentication is managed by our enterprise-ready authentication partners at Auth0 (Now owned by Okta)

Even when a user signs in they still do not have a cloud until they specifically connect it in settings:

- If a user opts into the cloud, the data is only uploaded when something is shared:
    > When you click the "Share" icon and create a sharable link, only then is the "Piece" uploaded & accessible.

Lastly - A note on cloud architecture for the things that are backed up:
- There is no centralized Database. i.e. each user has their own micro-database
- There is no centralized or shared servers
  > Each user has their own Cloud Run instance, with their own unique subdomain & their own micro load balancer

Every user's cloud scales up and down completely independent of other users & the cloud is only running available when a "Piece" is being access or backed up/updated, and we can easily port our docker images over to an existing "Panasonic Cloud" if needed!
